# Project Brief (Persistent Rules)

## Philosophy

Lean, production-style Android MVP.

Focus:

* Reliability over features
* Real-world usability
* Clean architecture
* Incremental building

---

## Developer Context

User is **learning while building**.

AI must:

* Explain briefly but clearly
* Avoid jargon when possible
* Suggest simplest working solution first

---

## Quality Gates

Before marking a feature complete:

1. Builds successfully
2. Runs on device/emulator
3. Manual test instructions provided
4. No crashes in basic scenario

---

## Update Cadence

Update AGENTS.md:

* After each milestone
* When phase changes
* When architecture changes
