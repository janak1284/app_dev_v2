# Project Updates - Location Alarm V2

## Predictive ETA Trigger (Dev 1)
- **Smart Alarms:** Implemented dual-trigger logic in `LocationAlarmService`. Users can now choose to be woken up based on a distance threshold OR a predictive ETA threshold (e.g., "Wake me 10 minutes before arrival").
- **UI Configuration:** Added a "Smart ETA Alarm" section to the configuration sheet with a toggle and minute slider.
- **Model Expansion:** Updated `AlarmSettings` to persist predictive alarm states.
- **Service Handshake:** Integrated the predictive parameters into the `MapViewModel` -> `LocationAlarmService` intent flow.
- **Service State Machine:** Refactored `LocationAlarmService` with a robust state machine (`TRACKING`, `ALARM_RINGING`). The service now remains active post-alarm for continuous breadcrumb recording.
- **Dynamic Re-routing Handshake:** Implemented a bi-directional loop where the service signals deviations (>100m) and `MapViewModel` pushes updated OSRM paths.
- **Spatial Math Engine:** Integrated `Turf-Java` for route-based distance calculation, route snapping, and cross-track deviation detection.
- **Predictive ETA:** Implemented a sliding window speed algorithm to provide real-time, road-aware ETAs.
- **Global Terminology Alignment:** Standardized on "Distance Alarm" and "Time Alarm" across the service and logic layers.

## Home Screen & Journey Summary Refinement (Dev 2)
- **Home Screen:** Overhauled the landing screen with a modern "Deep Dark" aesthetic, featuring a dynamic header and rich `RouteCard` components displaying journey metadata.
- **Journey Summary Sheet:** Replaced the basic save dialog with a polished `ModalBottomSheet` that triggers upon journey completion, allowing users to gracefully save or discard their routes.
- **Navigation Flow:** Unified the transition from journey tracking back to the saved routes list.
- **Visual Consistency:** Standardized colors and component shapes across the V2 UI layer.

## Technical Foundation
- **Room Migration:** Successfully migrated to KSP and updated to Room `2.7.0-rc01` for modern JVM compatibility.
- **Retrofit Client:** Dual-service configuration for Photon (Search) and OSRM (Routing).
- **Lifecycle Management:** Fixed memory leaks related to `BroadcastReceiver` unregistration and ensured foreground service persistence.
